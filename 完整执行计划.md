## 三、阶段一：数据补全（详细规则）

### 3.1 执行步骤

**步骤1: 分析缺失数据特征**
```sql
-- 查看缺失全文的信件特征
SELECT id, company_name, issue_date, url 
FROM warning_letters 
WHERE full_text IS NULL OR full_text = ''
LIMIT 50;
```

**步骤2: 尝试FDA API爬取**
- 使用FDA警告信搜索API
- 接口: `https://www.fda.gov/api/...`
- 每次请求间隔1秒
- 失败3次后跳过该记录

**步骤3: 尝试网页爬取**
- 如果API失败，尝试爬取网页
- 使用httpx库
- 设置User-Agent: `Mozilla/5.0`
- 每次请求间隔0.5秒

**步骤4: 数据清洗**
- 去除HTML标签
- 去除多余空白
- 保留段落结构

**步骤5: 入库更新**
```sql
UPDATE warning_letters 
SET full_text = ?, full_text_clean = ?
WHERE id = ?;
```

**步骤6: 触发翻译**
- 新增全文的信件自动触发翻译
- 使用Gemma 4 26B模型
- 翻译完成后自动评分

### 3.2 失败处理规则

| 失败类型 | 处理方式 |
|----------|----------|
| 网络超时 | 重试3次，间隔5秒 |
| 403禁止 | 跳过，记录日志 |
| 404未找到 | 跳过，标记为"无法获取" |
| 解析失败 | 跳过，记录原始内容 |
| 全部失败 | 标记状态为"failed"，不再重试 |

### 3.3 进度报告规则

- 每爬取50封报告一次进度
- 完成后生成最终报告
- 报告内容: 成功数、失败数、失败原因统计

### 3.4 质量标准

```
2. 主要违规事实
3. FDA要求的整改措施
4. 风险等级评估

警告信内容：
{translation}
```

**失败处理**: 超时或失败跳过，不阻塞流程

### 5.2 风险评级

**评级规则**:
- **高风险**: 涉及药品/医疗器械CGMP违规
- **中风险**: 涉及食品/烟草违规
- **低风险**: 其他类型违规

**评级逻辑**:
```python
def assess_risk(violations):
    high_risk_keywords = ['CGMP', 'drug', 'device', 'pharmaceutical', 'medical']
    medium_risk_keywords = ['food', 'tobacco', 'import']
    
    for v in violations:
        if any(kw in v.lower() for kw in high_risk_keywords):
            return 'high'
        if any(kw in v.lower() for kw in medium_risk_keywords):
            return 'medium'
    return 'low'
```

### 5.3 质量标准

- 摘要生成率 ≥ 90%
- 风险评级覆盖率 = 100%
- 摘要长度: 100-300字
