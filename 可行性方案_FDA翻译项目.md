# FDA警告信翻译项目可行性方案

## 📊 一、现状分析

### 1.1 数据总览

| 指标 | 数值 | 占比 |
|------|------|------|
| FDA警告信总数 | 986封 | 100% |
| 已获取全文内容 | 258封 | 26.2% |
| 已翻译 | 258封 | 26.2% |
| **待获取全文** | **728封** | **73.8%** |
| 待翻译 | 728封 | 73.8% |

### 1.2 翻译质量分布（258封已翻译）

| 等级 | 数量 | 占比 | 说明 |
|------|------|------|------|
| **A** (优秀) | 12封 | 4.7% | 可直接使用 |
| **B** (良好) | 52封 | 20.2% | 基本可用，轻微瑕疵 |
| **C** (合格) | 52封 | 20.2% | 需人工审核 |
| **D** (较差) | 49封 | 19.0% | 需要重翻 |
| **F** (不合格) | 93封 | 36.0% | 必须重翻 |

**关键发现：**
- 高质量率（A+B）：仅 **24.9%**
- 需重翻率（D+F）：高达 **55.0%**
- 总问题信件：**246封**（258封中有问题，占比 95.3%）

### 1.3 技术架构

- **翻译脚本**：`batch_translate_v2.py`（NVIDIA NIM API）
- **模型池**：8个模型轮询（DeepSeek、LLaMA、Nemotron、Mistral、Qwen）
- **质量校验**：`validate_translation.py`（4维度评分：长度比30%、AI词汇25%、导航残留25%、段落一致性20%）
- **数据库**：SQLite（`/root/data/fda_warning.db`）
- **并发**：5线程并行翻译

## 🚧 二、核心问题识别

### 问题1：数据缺口严重（优先级：🔴 最高）

**728封信件（73.8%）没有全文内容**，这是当前最大的瓶颈。

原因分析：
- FDA官网部分警告信以PDF形式发布，非HTML格式
- 部分信件可能已被归档或移除
- 爬虫程序可能未能成功抓取全文

**影响**：即使翻译质量完美，也只能覆盖26.2%的信件。

### 问题2：翻译质量不达标（优先级：🟡 高）

55%的翻译被评为D/F级，主要原因：

1. **AI总结词污染**（扣25分）：模型倾向添加"总之"、"综上所述"等总结性词汇
2. **导航残留**（扣15分）：FDA网页导航元素未被完全清洗
3. **段落结构破坏**（扣20分）：翻译后的段落数与原文不一致
4. **长度异常**（扣30分）：翻译过短或过长

### 问题3：并发进程冲突（优先级：🟠 中）

监控发现两个进程同时运行 `batch_translate_v2.py`：
- PID 256931：主进程
- PID 256859：重复进程

可能导致：
- API配额浪费
- 数据库写入竞争
- 翻译结果不一致

## 💡 三、解决方案

### 方案A：修复现有流程（推荐 ✅）

**目标**：在现有架构上优化，快速提升质量和覆盖率

#### 步骤1：数据补全（预计3-5天）

```
行动项：
1. 审查FDA官网爬虫脚本，修复PDF下载逻辑
2. 对728封缺失全文的信件，尝试重新抓取
3. 对无法抓取的信件，标记为"全文不可用"
4. 更新数据库，补充full_text字段
```

**预期结果**：覆盖率从26.2%提升至80-90%

#### 步骤2：翻译质量优化（预计2-3天）

```
行动项：
1. 优化翻译Prompt，强化"禁止添加总结"指令
2. 增加FDA术语专用词典（保持一致性）
3. 调整温度参数（当前0.1，可尝试0.05）
4. 对F级翻译批量重翻（使用更强模型）
```

**预期结果**：A+B级占比从24.9%提升至60%+

#### 步骤3：进程管理（预计1天）

```
行动项：
1. Kill重复进程（PID 256859）
2. 添加进程锁机制，防止重复启动
3. 实现优雅停止信号
```

**预期结果**：消除并发冲突

#### 资源需求

| 资源 | 用途 | 预估消耗 |
|------|------|----------|
| NVIDIA NIM API | 翻译调用 | ~5000次调用（728封×7模型轮询） |
| 时间 | 数据补全+重翻 | 5-8天 |
| 存储 | 临时文件 | <1GB |

---

### 方案B：更换翻译引擎（备选）

**目标**：使用更强模型（如GPT-4o、Claude）提升质量

```
优势：
- 翻译质量显著提升（预计A+B级达70%+）
- 更好的指令遵循能力

劣势：
- 成本大幅增加（GPT-4o约$0.03/千token，728封约$50-100）
- API配额限制
- 需要重构脚本
```

**适用场景**：预算充足、对质量要求极高

---

### 方案C：人工审核辅助（可选）

**目标**：建立人工审核机制，确保最终质量

```
流程：
1. AI翻译完成 → 自动分级
2. D/F级 → 人工重翻
3. C级 → 人工校对
4. A/B级 → 直接使用

预计人工成本：5-10小时（按100封重翻计算）
```
